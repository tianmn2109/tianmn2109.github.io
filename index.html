算法部分

算法在面试中是重点，而且基本上是手写代码，所以在刷题的时候一定不要在IDE中写代码，在记事本或者在网页上提供的编辑器内写代码，当然最好是先在白纸上写代码，然后在再输入到电脑上提交。国内公司的面试有的写算法有的不写算法只问项目，大部分是算法能写个大概就行，感觉国内公司的面试官主要看算法思想，然后差不多能实现就行。而外企的面试一般就只问算法，一般一轮面试45分钟，而且每场时间卡的很死，每场面试最好能写三道题，只写出来一道题就危险了。我在微软的mentor跟我说，除了博士毕业生他们会问背景外，其他的他们都只问算法。写算法他们关注的点一个是能不能做出来题目，给出的解是否是最有解，在面试者想不出来方法或者想出来的解不是最优的时候，他们会不断的给面试着提示，想通过这个来评价面试的潜力，还有就是聪明的程度。另一个就是面试者写出的代码的质量，是否有修改，是否考虑边界条件，特殊测试案例有没有考虑全，有没有语法错误，代码是否能直接编译运行。所以在面试外企时一般先将自己解题的方法跟面试官说，然后面试官觉得你的思想可以就会让你写代码，而如果面试官觉得思想错误或者可以优化就会给你提示，千万别不跟面试官沟通就直接写代码，这样万一写的不对时间也浪费了。在刷题时一定要练习编写代码的一次正确率。至于如何在一张白纸上写代码，hiredintech给出了一种方法，网站上有视频讲解，可以按照他们的方法去练习每到算法题。下面列出的是刷题的一些网站和书籍。


leetcode 这个是最重要的刷题网站，现在有两百多道题,一定要提前刷这个网站的题目，最好能多刷几遍，在做完每道题目的时候，看看discuss部分别人的解法，有些人的方法比较好，有些人的代码比较简练，可以在这个部分草看别人的代码。此外也可以看看自己代码的运行时间与别人运行时间的比较，找出自己算法比别人算法慢的原因。我的github上有我刷leetcode的代码，而且也把discuss部分别人较好的代码放在每个题目的文件夹下面。leetcode官方出版了一些题目解法代码的电子书Clean Code Handbook，这本电子书可以在我的github上找到。


careercup 这个网站上面大多是别人写的面经，里面会有不少算法题，这个网站整理出了一本书叫CC150（这本书也很重要）,这本书在国内翻译好像叫做程序员面试金典。CC150这本书在我的github上有。此外这个网站好像需要翻墙。


geeksforgeeks 这个网站的算法部分有不少leetcode上面没有的题目，而且对算法有较为详细的解释,里面的算法是分类的而且有讲解很不错。


lintcode 这个是九章算法他们弄得刷题网站，题目也很不错，里面有些书上的题目比如剑指Offer，所以可以在这个网站刷剑指offer的题目


july的博客也还行，上面整理了一些题目，不过里面的讲解什么的可能不全对，毕竟这个人水平不咋的，他的csdn的博客结构之法算法之道访问量很多，而且他后来在他的github上和一些人将这些题目整理成电子书，现在好像已经出版了，里面的算法题一般，但是里面他们总结的解决海量数据的方法可以参考下，很容易，而且在面试时挺有用的。另外他还开了个培训班，有同学买过他们讲解算法的课程，我这有，需要的话我通过网盘分享给你们，不过我没看过这些视频，不知道讲的咋样。


算法书：CC150， Clean Code Handbook, 编程之美，挑战程序设计竞赛， 算法导论， 计算机算法设计与分析(王晓东)， 算法设计， C++标准程序库，STL源码剖析,建议买纸质版书看，电子版在我的github上整理了。


编程语言

语言这个东西在国内面试的时候也会被问道，一般会C/C++就行了，而且算法部分的话最好也用C++写，所以需要好好深入学习C++。语言问的深的话就是一些语法，比如某些关键字的作用，或者是实现机制如需函数如何实现的。这个相对来说比较容易准备。建议看一下C++标准程序库和STL源码剖析，这样会对C++有更好更全面的了解。下面列一下书籍博客和知识点，可能比较乱。

书籍：C和指针，C缺陷与陷阱，effectiveC++，C语言深度解剖（这个可以看作是前几本书的总结，但是里面也有不少错误，这本书针对的环境是VC6.0,没考虑gcc等环境，不过还是不错的这本书），C++标准程序库（看这本感觉可以不用看C++Primer），C++源码剖析（这个强烈建议看，对着源代码），深度探索C++对象模型（这本书主要是C++内部的实现机制）。上面这些书绝大部分的电子版在我的github上都有。


geeksforgeeks这个网站的语言部分，他们总结了C和C++各种可能被问到的知识点，强烈建议看这个，这个看完应该对这两门语言的理解更加深刻。geeksforgeeks中关于C和C++的总结在我的github上的 classify_and_review/geeks_for_geeks_c++文件中上有一些，但是这些是我记录下来感觉自己可能会忘记的知识点，所以很不全。

系统设计题

系统设计题在国内的公司问的比较少，我只有在面今日头条的时候被问过如何设计他们的推荐系统，还有就是百度外卖面试官问的设计一个通讯录(这个确切说算设计提)。这种题最难准备，没有固定的答题方式，我看的面经系统设计题外企会问，我在面微软实习生的时候最后一面就被问了系统设计题（大概是他们每天有源源不断的搜索关键词，让你设计一个系统，刚开始需求很不明确，说了一点之后，面试官就不断加条件，如数据量非常大，单台机器无法存储，如何统计24小时内前100名的搜索词等等等等），当时一点大体思路都没有，所以建议还是准备准备。系统设计题的大体方式强烈建议看看hiredintech的系统设计题的讲解，看完这个讲解应该很有感触，系统设计题目的主要点是与面试官沟通，面试官给的题目的条件很笼统，所以要不断于面试官沟通，将问题具体化，数据化。这个完了之后建议看看carrercup上面别人的面经，会提到很多系统设计题目，一般别人都会有些解答。书的话建议看大型网站技术架构，通过这本书大概了解下会设计到的东西。

网站：careercup在这个网站选择过滤选项，搜索出系统设计题或者设计题，可以找到很多题目

Code

代码的质量非常重要，尤其在面外企的时候，可以看看google编程规范和编写可读代码的艺术。主要的还是练习，先看这个在刷算法。

Linux

这个也是很重要的，因为以后在公司写程序基本都是在linux下，linux下常见的命令，一些基本的远离，还有就是awk这个要好好掌握，所以建议以后写程序就在linux下写，调试用gdb调试，告别windows。

网络，操作系统和数据库

面试时还问问数据库，网络，操作系统的一些知识，问的都比较简单，比如写sql语句，三次握手，线程通信方式等，这些东西网上一大堆，以后有空在整理这部分。


项目和实习

项目是国内公司面试的重点，也是我们的弱点，一个是项目少，二是项目跟互联网不相关，我当时就被好几个面试官问问什么转行做互联网，面360的时候投的是服务器端开发，结果那面试官调戏我，比如没做过这方面的项目为什么报这个职位，解释了一下然后又说你为这个职位准备了什么，为什么不去做点关于这个职位的开源项目等等，现在想想就来气，当时一上来就让我给他将个项目，我将毕业设计的，估计他没听懂，讲了一会就把我打断为我报什么职位，然后问我做了多长时间我说10个月，然后他就说一个项目做了十个月时间也太长了。总之感觉就是因为方向什么的被调戏了。

我只能说实习很重要，除非在实验室做的项目特别牛而且与他们相关。建议如果找实习的话最好是跟以后工作的职位相关的。


简历

简历的书写感觉要突出项目，比如项目是做什么的，自己采用了什么方法，达到了什么效果，比别人好多少，最好能用数字刻画。简历一页就够了。现在也可以写写自己的简历，感觉自己简历没啥写的时候就想想到找工作的时候想简历上有什么东西，然后就知道自己该做那些事情了。当时我就想简历要由三部分组成，实习，开源代码和学校的项目，然后就是按照这个准备的。简历的话我的github上有我自己和一些同学的，你们可以参考下。英文简历的话careercup上面提供了一个很好的模板，而且标出每部分该怎么写，挺好的，中文的也可以借鉴人家的思想。另外glassdoor网站注册帐号后，可以提交简历，他们会有专门的人联系你帮你改简历。
